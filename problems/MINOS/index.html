MINOS Maze generator

Minos, the king of Crete, wants to build labyrinth for his son Minotaur. The labyrinth should have just one floor and will be digged in underground from one entrance. Junctions of the labyrinth should form a square grid and there should be exactly one way between every two junctions. The king wants to minimize sweat of labourers, he wants to minimize total hardness of mined material and minimize transport path to the entrance.
Write a program that generates a maze on the grid NxM. The digging is performed on one place at time and after approaching the new junction the hardness (0 up to 255) is assigned to posible outgoing corridors. This is done in order (0-east, 1-west, 2-north, 3-south) using the congruential generator:

A_i = B_i mod 256,

B_i = B_i * 1664525 + 1013904223 mod 2^32,

skipping the corridors that have already assigned hardness. The value B_0 is part of the input. The entrance into the maze is in the top left corner from the left side.

Input

One line with numbers N, M and B_0.

Output

The maze as table of M character on N lines. Every junction is coded by one hexadecimal digit, where the bits on positions (0-east, 1-west, 2-north, 3-south) are one if there is outgoing corridor to the particular direction.

Example


Input:
3 3 3

Resulting maze:

  X  - 3 - X - 45 - X
  |                  
  134      168      231
  |         
  X - 26 - X - 92 - X
           |        |
  177      11       80
           |        | 
  X - 117- X   238  X

Output:
932
5BA
164
Limits
Time limit
60s
